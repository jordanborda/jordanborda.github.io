Big O Notation in JS

by Jordan Piero Borda Colque

Big O notation se centra en el peor de los casos y se puede utilizar para describir el tiempo/espacio de ejecución mediante un algoritmo. Exploremos ahora los tipos más comunes de notaciones Big O. Usaremos ejemplos de Vanilla JS y ECMA6.

Para ilustrar los resultados, usaremos el tiempo (milisegundos - ms) y el espacio (tamaño de la matriz).

Probar configuraciones de entradas de datos

Para probar el espacio de datos utilizando una matriz de 3 con diferentes tamaños: 10,000, 10,0000 y 1,000,000 elementos.

Por tiempo usando "console.time" y "console.timeEnd", lo que le permite rastrear el tiempo de ejecución en MS. Cada caso se puede copiar/pegar en la terminal de la consola y jugar con las respuestas.

O(1) - Constant time

O(1) siempre se ejecuta al mismo tiempo (o espacio). Por ejemplo, busque el primer elemento de la matriz. La matriz de entrada podría ser de 1 elemento o de 10 000, 10 0000 y 1 000 000 de elementos, pero este método solo requeriría un "paso".

Todo el tiempo de ejecución será inferior a 0,01 ms.

O(N) - Linear Time Algorithm

El peor de los casos para los datos de prueba es invertir todos los elementos de la matriz de principio a fin, por lo que requiere N iteraciones de bucle. La función de caso de prueba busca por valor desde el final de la matriz hasta el comienzo.

O(N2) - Quadratic Time Algorithm

O(N2) representa un algoritmo cuyo rendimiento es el cuadrado del número de entradas.El tiempo del algoritmo de ordenación por selección crece exponencialmente en relación con el número de entradas.

O(2N)

O(2N) un algoritmo cuyo crecimiento se duplica con cada adición al conjunto de datos de entrada. El famoso ejemplo es el cálculo recursivo de Fibonacci number.

O(log n) - Logarithms

O(log n) la mayoría de los algoritmos de clasificación más rápidos como QuickSort, Merge-sort. Echemos un vistazo al ejemplo del algoritmo de búsqueda binaria. Para datos de prueba, use entradas 10,000, 100,000 y 1,000,000. Para generar elementos aleatorios de una matriz usando la biblioteca Math: arr[Math.floor(Math.random() * arr.length)] La búsqueda binaria se utiliza para buscar datos de entrada ordenados seleccionando el elemento medio del conjunto de datos, esencialmente la mediana, y lo compara con un valor objetivo. Si los valores coinciden, devolverá el éxito.

Aumentar el tamaño del conjunto de datos de entrada tiene poco crecimiento. Esto hace que los algoritmos O(log n) sean muy efectivos para grandes conjuntos de datos.
